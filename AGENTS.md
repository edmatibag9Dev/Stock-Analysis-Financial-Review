This file is the canonical entry point for any AI agent (Claude Code, Cowork, Codex, etc.)
asked to **use, reference, extend, or rebuild** this project. Read it before acting.

## What this repo is

A fundamental + technical investment-analysis workflow for publicly traded stocks. For each ticker it
produces a professional investment memo (`.docx`) and, where applicable, a valuation Excel model
(`.xlsx`), generated by per-ticker build scripts. The workflow itself is captured in
`stock-analysis.skill` and `CLAUDE.md`.

Design in one line: **one dated folder per analysis (`{TICKER}-{YYYY-MM-DD}/`), each holding the memo,
the model, and the build script that generated them.**

## The data contract (per-analysis folder convention)

Every analysis is one folder named `{TICKER}-{YYYY-MM-DD}/`. Consumers (and the README index table)
rely on that name to locate an analysis by ticker and date. Each folder contains:

```
{TICKER}-{YYYY-MM-DD}/
  {TICKER}_Investment_Memo.docx     # 10-section memo — verdict + DCF bull/base/bear
  {TICKER}_..._Model.xlsx           # valuation model (when the ticker warrants one)
  build_model.py | create_*.js      # the generator for the above, kept beside its output
```

Rules an agent must preserve:
- **Never overwrite a dated analysis.** A re-run for the same ticker on a new date gets a NEW
  `{TICKER}-{YYYY-MM-DD}/` folder; the prior one is history (move superseded ones to `{TICKER}_Archive/`).
- **The memo is 10 sections** (business snapshot, bull/bear, DCF, Rule of 40 for SaaS, peer comps,
  technical setup, options overlay, verdict); the model is the 4-sheet layout above.
- **DCF is 3-scenario (bull/base/bear) with a single WACC**; EBITDA-terminal for pre-profitable names.
- Keep the README's analyses table in sync (ticker, company, date, rating, DCF bull/base/bear).

## How it works

1. Invoke the workflow (`stock-analysis.skill`) for a ticker.
2. Research: SEC 10-Q/8-K filings, guidance, unit economics; technicals (SMA 20/50/200, RSI, IV rank).
3. The build script generates the memo `.docx` (+ model `.xlsx`) into a fresh `{TICKER}-{YYYY-MM-DD}/`.
4. Update the README analyses table with the verdict and DCF scenarios.

## How to extend

- **New analysis:** create `{TICKER}-{YYYY-MM-DD}/`, run/author its build script, add a README row.
- **Change memo/model structure:** update `stock-analysis.skill` + `_Analysis_Patterns/` so every future
  analysis inherits it; do not retro-edit archived deliverables.
- **Prerequisites** for the build scripts are documented in the README ("Running the Build Scripts").

## Privacy — hard rules

- **Public repo.** Analyses are Ed's own research on public equities — not investment advice. Never
  commit brokerage credentials, account numbers, position sizes, or any personal/financial PII, and
  never any API key. Keep only public-market analysis in the deliverables.
- If a build script needs a data-provider key, read it from an env var; never hard-code or commit it.

## Verification gates (run before declaring a change done)
1. The new analysis lives in a correctly named `{TICKER}-{YYYY-MM-DD}/` folder; no prior folder overwritten.
2. Memo has all 10 sections; model has its 4 sheets (where a model applies).
3. README analyses table updated (ticker, company, date, rating, DCF bull/base/bear).
4. No secrets/PII committed (grep the diff for keys/account numbers).
5. Commit follows `CONTRIBUTING.md`; README updated on feat/data.
